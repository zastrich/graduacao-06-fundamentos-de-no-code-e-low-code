# 🎬 Roteiros de Vídeos: TeamFlow (No-Code)

Este documento contém os roteiros estruturados para a gravação dos vídeos complementares do projeto prático da disciplina de **Fundamentos de No-code e Low-code**.

---

## Vídeo 1: Apresentação do Projeto (Pitch e Visão Geral)
**Objetivo:** Apresentar a dor (problema), a solução (TeamFlow) e os benefícios da abordagem No-Code.
**Duração estimada:** 2 a 3 minutos.

### Estrutura do Roteiro

| Tempo | Tela / Visual | Fala / Áudio Sugerido |
| :--- | :--- | :--- |
| 0:00 - 0:30 | **Câmera no(a) aluno(a)** (Rosto) ou **Slide de Capa** com o nome do projeto (TeamFlow) | "Olá! Meu nome é [Seu Nome], e este é o meu projeto prático para a disciplina de Fundamentos de No-Code e Low-Code. O projeto que desenvolvi se chama **TeamFlow**." |
| 0:30 - 1:00 | **Slide ou Animação simples** mostrando o problema (excesso de planilhas, falta de comunicação em equipes). | "O principal problema que o TeamFlow busca resolver é a desorganização e a perda de informações no gerenciamento de equipes. Muitas vezes, empresas usam diversas ferramentas separadas para tarefas, comunicação e alertas, o que gera inconsistência." |
| 1:00 - 1:45 | **Gravação de Tela (Screencast)** mostrando o Dashboard principal do TeamFlow (Quadro Kanban). | "Como solução, criei o TeamFlow utilizando a ferramenta Bubble.io. É uma plataforma centralizada e de interface amigável. Nela, o time pode visualizar todas as atividades em um quadro Kanban escalável, separar tarefas por status – como 'A Fazer', 'Em Progresso' e 'Concluído'." |
| 1:45 - 2:30 | **Câmera no(a) aluno(a)** ou **Slide de Encerramento**. | "O grande diferencial de usar uma abordagem No-Code como o Bubble foi a velocidade de iteração. Consegui modelar o banco de dados e aplicar lógicas complexas visualmente em uma fração do tempo que levaria com código. Muito obrigado(a)!" |

## Vídeo 2: Demonstração Técnica e Funcionalidades (Showcase)
**Objetivo:** Demonstrar o sistema funcionando, o banco de dados visual no Bubble e as automações (Workflows) configuradas.
**Duração estimada:** 4 a 5 minutos.

### Estrutura do Roteiro

| Tempo | Tela / Visual | Fala / Áudio Sugerido |
| :--- | :--- | :--- |
| 0:00 - 0:45 | **Gravação de Tela:** Editor do Bubble na aba `Data`, mostrando os `Data Types` criados/traduzidos. | "Neste segundo vídeo, vou aprofundar na estrutura técnica do TeamFlow. Começando pela Modelagem de Dados Visual no painel do Bubble. Criei tipos relacionais, onde as 'Tasks' (Tarefas) estão vinculadas diretamente aos 'Users' e 'Teams'. Traduzimos todas essas estruturas para o Português." |
| 0:45 - 1:30 | **Gravação de Tela:** Editor do Bubble na aba `Privacy`. | "A segurança e controle ético dos dados são requisitos mandatórios. Através das 'Privacy Rules', impusemos que os dados sigilosos dos usuários sejam visíveis apenas sob condições específicas, protegendo toda a integridade do sistema que estamos gerindo." |
| 1:30 - 2:45 | **Gravação de Tela:** Aplicação rodando no modo **Preview**. Navegar e criar uma tarefa. | "Agora, vamos usar a aplicação final. Vou simular a criação de uma nova tarefa no nosso Kanban e atribuí-la a Camila Ferreira. Aqui demonstramos a responsividade e o vínculo de Data Sources diretamente nos campos. Tudo que modifico salva no banco imediatamente." |
| 2:45 - 3:45 | **Gravação de Tela:** Editor do Bubble na aba `Workflow` e demonstração das Notificações. | "Essas ações geram reações em cadeia controladas pela aba de Workflow. Por exemplo, ao atualizar o registro de Status para 'Concluído', temos uma rotina configurada para atualizar a Data de Modificação daquele registro interno de forma automática." |
| 3:45 - 4:15 | **Câmera no(a) aluno(a)**. | "Assim, cobrimos todos os requisitos práticos exigidos no projeto, consolidando tanto as interfaces interativas quanto o robusto backend relacional no ecossistema No-Code do Bubble. Agradeço por assistirem!" |

*Dicas do Estudante (Checklist de Gravação):*
